OOP CORE - C#

🧱 1. Trừu tượng (Abstraction)
"Chỉ cần biết chức năng, không cần biết chi tiết bên trong"

Trừu tượng là cách ta "giấu" đi những chi tiết phức tạp và chỉ cung cấp ra bên ngoài một giao diện đơn giản. Trong C#, bạn có thể sử dụng abstract class hoặc interface để đạt được điều này.

Phân biệt abstract class và interface:
- Abstract Class: Dùng khi các lớp có mối quan hệ "is-a" (là một) và chia sẻ code. Nó có thể chứa phương thức đã có code, thuộc tính và constructor. Một lớp chỉ có thể kế thừa từ một abstract class duy nhất.
- Interface: Dùng để định nghĩa một "hợp đồng" (contract) mà các lớp không liên quan có thể cùng tuân thủ. Nó mô tả khả năng "can-do". Trước C# 8.0, interface không chứa code; từ C# 8.0 trở đi, nó hỗ trợ default implementation (triển khai mặc định). Một lớp có thể thực thi nhiều interface.

Ví dụ trong đề tài Grab:
- Bạn có thể tạo một abstract class User để đại diện cho một người dùng chung chung trên hệ thống (có UserId, PhoneNumber, Name).
- Từ đó, bạn tạo các lớp cụ thể là Driver (Tài xế) và Customer (Khách hàng) kế thừa từ User.
- Như vậy, bạn đã trừu tượng hóa khái niệm "Người dùng".

🔒 2. Đóng gói (Encapsulation)
"Bảo vệ dữ liệu, kiểm soát quyền truy cập"

Đóng gói là cơ chế để bảo vệ dữ liệu của một đối tượng khỏi sự truy cập trực tiếp từ bên ngoài, và chỉ cho phép thay đổi dữ liệu đó thông qua các phương thức được chỉ định. Điều này được thực hiện thông qua các Access Modifier (từ khóa điều khiển truy cập) như private, public, protected, internal.

Ví dụ trong đề tài Grab:
- Trong class Driver, bạn có thể có một thuộc tính private bool IsOnline (Tài xế có đang rảnh không). Bạn sẽ không cho phép bên ngoài tự ý gán true hoặc false.
- Thay vào đó, bạn cung cấp các phương thức public void AcceptRide() và public void CompleteRide() để kiểm soát và thay đổi trạng thái IsOnline một cách hợp lệ.
- Việc class Customer không thể tự ý thay đổi Rating (Đánh giá) của Driver cũng là một ví dụ của tính đóng gói.

👨‍👧‍👦 3. Kế thừa (Inheritance)
"Tái sử dụng và mở rộng chức năng"

Kế thừa cho phép bạn tạo ra một lớp mới dựa trên một lớp đã có. Lớp mới sẽ được thừa hưởng tất cả các thành phần của lớp cũ, và có thể thêm hoặc sửa đổi chúng.

Ví dụ trong đề tài Grab:
- Driver và Customer kế thừa từ abstract class User.
- Bạn cũng có thể tạo ra các loại Vehicle khác nhau: XeMay (Xe máy), XeHoi (Xe hơi) kế thừa từ abstract class Vehicle.

🎭 4. Đa hình (Polymorphism)
"Cùng một hành vi, nhiều cách thể hiện"

Đa hình (nhiều hình dạng) cho phép các đối tượng thuộc các lớp khác nhau có thể được xử lý thông qua cùng một giao diện, và mỗi đối tượng sẽ có cách thực thi hành vi riêng của mình.

Đa hình có hai loại chính:
1) Compile-time Polymorphism:
- Method Overloading (Nạp chồng phương thức)
- Operator Overloading (Nạp chồng toán tử)

Ví dụ:
- Nạp chồng phương thức TinhCuocPhi trong class Ride để nhận tham số là distance (double) hoặc pickup và destination (Location).

2) Run-time Polymorphism:
- Method Overriding (Ghi đè phương thức)
- Dùng từ khóa virtual ở lớp cha và override ở lớp con.

Ví dụ:
- Định nghĩa virtual double CalculatePrice() trong class Ride.
- Tạo các lớp con như StandardRide và SurgeRide, mỗi lớp override CalculatePrice() với công thức tính giá riêng.
- Khi gọi ride.CalculatePrice(), chương trình tự động biết cần chạy công thức nào.

OOP: Classes, Objects và C#

1. Lập trình hướng đối tượng (OOP)
OOP = phương pháp lập trình dựa trên Class và Object.

Ý tưởng:
- Chương trình được xây dựng bằng cách mô phỏng các đối tượng trong thế giới thực.

Ví dụ:
- Thực tế: Nhân viên → object NhanVien; Xe ô tô → object XeOto; Hóa đơn → object HoaDon
- Trong chương trình: class tương ứng mô phỏng các thực thể đó.

2. Object và Class
- Object = thực thể cụ thể
  Ví dụ: SinhVien sv1, XeOto car1, NhanVien nv1
- Class = bản thiết kế của object
  Ví dụ: Class: NhanVien; Object: nv1, nv2, nv3

3. Thành phần của Class
Thành phần:
- Field: biến lưu dữ liệu
- Property: truy cập dữ liệu
- Method: hàm xử lý
- Event: thông báo sự kiện

Ví dụ class:
class NhanVien
{
	public string Ten;
	public double HeSoLuong;
 	
	public double TinhLuong()
	{
    	return HeSoLuong * 1500000;
	}
}

4. 4 đặc trưng của OOP
- Abstraction: trừu tượng hóa
- Encapsulation: đóng gói dữ liệu
- Inheritance: kế thừa
- Polymorphism: đa hình

Ví dụ:
Abstraction
- Tạo class chung
  class Xe

Encapsulation
- Ẩn dữ liệu bằng private
  private double luong;

Inheritance
- class XeDien : Xe

Polymorphism
- TinhLuong()
- TinhLuong(int bonus)

5. Khai báo Class trong C#
Cú pháp:
public class TenClass
{
	// fields
	// properties
	// methods
}

6. Tạo Object

Hoặc
NhanVien nv1;
nv1 = new NhanVien();

Lưu dữ liệu của object.

Cho phép truy cập field private

	set { heSoLuong = value; }

9. Method (Phương thức)
Là hàm xử lý của class.
public double TienThuong()
{
	return 1000000;
}

10. Constructor
Constructor = hàm khởi tạo object.
- Không tham số:

	{

	}

- Có tham số:

Dùng để hủy object.

{
	~Car()
	{
    	// cleanup
	}

Nhiều hàm cùng tên nhưng khác tham số

- Truyền tham trị (value)

Không thay đổi biến ngoài.

Biến ngoài bị thay đổi.

Inheritance và Polymorphism (C#)

1. Tính thừa kế (Inheritance)
Khái niệm:
- Inheritance = cho phép class con (derived class) kế thừa thuộc tính và phương thức của class cha (base class).
Mục đích:
- Tái sử dụng code
- Mở rộng chức năng

Ví dụ:
class Xe
{
	public string Hang;
 	
	public void Chay()
	{
    	Console.WriteLine("Xe đang chạy");
	}
}

class XeDien : Xe
{
	public int DungLuongPin;
}

Quan hệ kế thừa:
- Class cha (Base class) → Class con (Derived class)

Ví dụ:
NhanVien ↓
NhanVienFullTime
NhanVienPartTime

2. Xây dựng lớp kế thừa
Cú pháp:
class LopCon : LopCha
{
}

Ví dụ:
class NhanVien
{
	public string Ten;
}

class QuanLy : NhanVien
{
	public int PhongBan;
}

3. Lớp trừu tượng (Abstract)
Khái niệm:
- Abstract class: không thể tạo object trực tiếp, dùng làm class cha.

Ví dụ:
abstract class Shape
{
	public abstract double Area();
}

Class con:
class Circle : Shape
{
	public double r;
 	
	public override double Area()
	{
    	return 3.14 * r * r;
	}
}

4. Lớp niêm phong (Sealed)
Khái niệm:
- Sealed class: không cho phép lớp khác kế thừa.

Ví dụ:
sealed class Database
{
}
Sai:
class MyDB : Database   // lỗi
{
}

5. Tính đa hình (Polymorphism)
Khái niệm:
- Polymorphism = một hành vi nhưng nhiều cách thực hiện khác nhau.

6. Hai loại đa hình
- Compile-time polymorphism: Method Overloading
- Run-time polymorphism: Method Overriding

7. Method Overloading
Cùng tên hàm nhưng khác tham số
class Math
{
	public int Cong(int a, int b)
	{
    	return a + b;
	}
 
	public int Cong(int a, int b, int c)
	{
    	return a + b + c;
	}
}

8. Method Overriding
Class con ghi đè phương thức của class cha
Class cha:
class Animal
{
	public virtual void Speak()
	{
    	Console.WriteLine("Animal sound");
	}
}

Class con:
class Dog : Animal
{
	public override void Speak()
	{
    	Console.WriteLine("Dog barks");
	}
}

9. Ví dụ đa hình
Animal a;

a = new Dog();
a.Speak();

a = new Cat();
a.Speak();
Kết quả:
Dog barks
Cat meows

Tóm tắt:
Inheritance: class Con : class Cha; Abstract: abstract class; abstract method; Sealed: sealed class; Polymorphism: virtual/override; Overloading vs Overriding.

Interface và Operator Overloading (C#)

1. Interface – Khái niệm
Interface = khai báo tập hợp phương thức/thuộc tính nhưng không có code thực thi.
Đặc điểm:
- chỉ khai báo, không implement
- lớp thực thi phải cài đặt toàn bộ phương thức
- là reference type

Ví dụ:
public interface IComparable
{
	int CompareTo(object obj);
}

2. Thực thi Interface
Class thực thi bằng :
class MyClass : IComparable
{
	public int TheValue;
 	
	public int CompareTo(object obj)
	{
    	MyClass mc = (MyClass)obj;
 
    	if (this.TheValue < mc.TheValue) return -1;
    	if (this.TheValue > mc.TheValue) return 1;
    	return 0;
	}
}

Ý nghĩa CompareTo:
- < 0: Nhỏ hơn
- > 0: Lớn hơn
- = 0: Bằng

3. Quy tắc Interface
- Interface: không chứa dữ liệu
- chỉ chứa: methods, properties, events, indexers, các thành phần mặc định public
- Tên thường bắt đầu bằng: I + Name
Ví dụ: IComparable, IDisposable, ICloneable

4. Ép kiểu Interface
Không thể gọi interface trực tiếp từ object nếu không ép kiểu.
MyClass mc = new MyClass();
IIfc1 ifc = (IIfc1)mc;

ifc.PrintOut("Interface");

5. Toán tử as
Ép kiểu an toàn.
IIfc1 ifc = mc as IIfc1;
Kết quả:
- Class implement interface → trả về object
- Không implement → trả về null

6. Một class thực thi nhiều interface
class MyData : IDataRetrieve, IDataStore
{
	int Mem1;
 	
	public int GetData()
	{
    	return Mem1;
	}
 
	public void SetData(int x)
	{
    	Mem1 = x;
	}
}

7. Interface kế thừa Interface
interface IDataRetrieve
{
	int GetData();
}
 
interface IDataStore
{
	void SetData(int s);
}
 
interface IDataIO : IDataRetrieve, IDataStore
{
}

8. Interface IDisposable
Interface:
public interface IDisposable
{
	void Dispose();
}
Mục đích:
- giải phóng unmanaged resources
- không giải phóng bộ nhớ managed
Ví dụ:
file handle
database connection
socket
Sau khi dispose: GC.SuppressFinalize(this); để ngăn finalizer chạy.

9. Interface ICloneable
Dùng để tạo bản sao object.
public interface ICloneable
{
	object Clone();
}
Ví dụ:
public object Clone()
{
	return new Student(this.id, this.name,
    	(Assessment)this.assessment.Clone());
}

Ý nghĩa: copy object; deep copy

10. Operator Overloading
Cho phép định nghĩa lại toán tử cho class.
Ví dụ:
+  -  *  /  ==  <  >

11. Cú pháp
Unary operator:
public static Type operator +(Type a)
Binary operator:
public static Type operator +(Type a, Type b)

12. Ví dụ – cộng phân số
public static PhanSo operator +(PhanSo a, PhanSo b)
{
	PhanSo kq = new PhanSo(0,1);
 
	kq.tu = a.tu * b.mau + a.mau * b.tu;
	kq.mau = a.mau * b.mau;
 
	return kq;
}

Sử dụng:
PhanSo a = new PhanSo(2,3);
PhanSo b = new PhanSo(4,3);

PhanSo c = a + b;

13. Các toán tử có thể overload
Unary:
+x  -x  !x  ~x  ++  --
Binary:
+  -  *  /  %
&  |  ^
<<  >>
==  !=
<  >  <=  >=

Không overload được:
&&
||
=
?.
new
typeof

14. Quy tắc
- operator phải static
- toán tử so sánh phải định nghĩa theo cặp
Ví dụ:
== phải đi với !=
< phải đi với >
<= phải đi với >=

Tóm tắt:
Interface: hợp đồng hành vi; Implement: class phải viết code cho method; Multiple interface: class có thể implement nhiều; IDisposable/ICloneable/Operator Overloading.

Delegate và Event (C#)

1. Delegate – Khái niệm
Delegate = kiểu dữ liệu tham chiếu đến phương thức.
Đặc điểm:
- giống function pointer
- type-safe
- hỗ trợ object-oriented

2. Delegate định nghĩa gì
Delegate định nghĩa signature của hàm:
- return type
- danh sách tham số
Ví dụ:
public delegate void MyDelegate1(int x, int y);
public delegate string MyDelegate2(float f);

Ý nghĩa:
- MyDelegate1 → method dạng: void Method(int,int)
- MyDelegate2 → method dạng: string Method(float)

3. Tạo instance delegate
void Method1(int x, int y)
{
	Console.WriteLine(x + y);
}

MyDelegate1 del = new MyDelegate1(Method1);

4. Gọi delegate
del(5, 10);
Delegate sẽ gọi method mà nó tham chiếu.

5. Multicast Delegate
Delegate có thể tham chiếu nhiều method.
Thêm method:
del += Method2;
Xóa method:
del -= Method1;
Ví dụ:
void Print(int x, int y)
{
	Console.WriteLine($"x={x}, y={y}");
}

void Sum(int x, int y)
{
	Console.WriteLine(x + y);
}

MyDelegate del = Print;
del += Sum;

del(5,10);
Kết quả:
x=5,y=10
15

6. Delegate dùng cho Sort tổng quát
Vấn đề:
Sort object bất kỳ → cần quy tắc so sánh
Giải pháp:
truyền hàm so sánh vòa Sort bằng delegate.

Delegate so sánh
public delegate bool CompareObj(object o1, object o2);

Sort tổng quát
public static void Sort(object[] objs, CompareObj cmp)
{
	for(int i=0;i<objs.Length-1;i++)
    	for(int j=objs.Length-1;j>i;j--)
        	if(cmp(objs[j],objs[j-1]))
            	Swap(objs[j],objs[j-1]);
}

Class tự định nghĩa cách so sánh
class Person
{
	public string name;
 
	public static bool CompareName(object p1, object p2)
	{
    	return string.Compare(((Person)p1).name,
                              ((Person)p2).name) < 0;
	}
}

Gọi Sort
CompareObj cmp = new CompareObj(Person.CompareName);

Lib.Sort(persons, cmp);

7. Event – Khái niệm
Event = cơ chế thông báo khi một sự kiện xảy ra.
Thành phần:
- Publisher: lớp phát sinh sự kiện
- Subscriber: lớp xử lý sự kiện

8. Ví dụ GUI
Button Click

Ngoại lệ (Exceptions) trong C# – Lập trình hướng đối tượng.

1. Khái niệm Code Path – Error – Exception
Code Path
- Là chuỗi các phương thức được gọi tuần tự khi chương trình chạy.
- Các lời gọi được lưu trên stack.
Ví dụ:

Error (Lỗi)
Lỗi xảy ra do nhiều nguyên nhân:
- Lập trình viên
- Sai cú pháp
- Sai thuật toán
- Người dùng
- Nhập sai định dạng
- Nhập vượt phạm vi
- Môi trường
- File không tồn tại
- Hết dung lượng đĩa
- Mạng lỗi

Error = Compile Error + Runtime Error

Exception
- Exception là lỗi xảy ra khi chương trình đang chạy (Runtime).
- Khi exception xảy ra, chương trình không thể tiếp tục thực hiện bình thường.
Ví dụ:
- Chia cho 0
- File không tồn tại
- Sai định dạng số

Exception = Runtime Error

2. Giải pháp xử lý lỗi
Trong C# có 3 cách chính:

- Kiểm tra dữ liệu (if(input != null))

- CLR tạo đối tượng Exception
- Exception được throw
- Exception truyền theo code path
- Tìm phương thức có catch
- Nếu không có catch → chương trình dừng
Ví dụ luồng:
Divide() → Three() → Two() → One() → Main()
Exception sẽ truyền ngược lại theo thứ tự trên.

4. Các lớp Exception thông dụng

- ArgumentException, ArgumentNullException, ArgumentOutOfRangeException

- Source
- StackTrace

6. Cú pháp xử lý Exception
try
{
   code có thể gây lỗi
}

   xử lý lỗi
}
finally
{
   luôn được thực hiện

SERIALIZATION & DESERIALIZATION (C#)

Serialization

Ví dụ:
Object → JSON / XML / Binary

Deserialization

JSON / XML / Binary → Object

2. Các loại Serialization trong C#
- Binary Serialization: BinaryFormatter, DataContractSerializer

- JSON Serialization: Newtonsoft.Json, System.Text.Json

3. Interface ISerializable
Để tự kiểm soát serialization, class có thể implement interface ISerializable.
Cấu trúc:
class Student : ISerializable
{
	public void GetObjectData(SerializationInfo info, StreamingContext context)
	{
    	info.AddValue("key", value);
	}
 
	public Student(SerializationInfo info, StreamingContext context)
	{
    	Property = info.GetValue("key", typeof(Type));
	}
}

4. Serialize Object
Dùng GetObjectData()
Ví dụ:
public void GetObjectData(SerializationInfo info, StreamingContext context)
{
	info.AddValue("Name", Name);

}

5. Deserialize Object
Dùng constructor đặc biệt
public Student(SerializationInfo info, StreamingContext context)
{
	Name = (string)info.GetValue("Name", typeof(string));
	Age = (int)info.GetValue("Age", typeof(int));
}

Observer Pattern = Event & Delegate

1. Observer Pattern = Event & Delegate (Lắng nghe và phản ứng)
Ý tưởng cốt lõi
- Publisher phát đi thông báo khi có sự kiện xảy ra.
- Nhiều Subscriber đăng ký lắng nghe và nhận thông báo mà không cần Publisher biết chi tiết.

Trong C#:
- delegate và event là cơ chế tương ứng.

Áp dụng vào Grab:
- Khách hàng tạo yêu cầu đặt xe → một sự kiện OnNewRideRequest được phát ra.
- Tất cả tài xế online sẽ nhận thông báo này.

Code minh họa:

csharp
// === PUBLISHER ===
public class RideDispatcher
{
    public delegate void RideRequestEventHandler(object sender, RideRequestEventArgs e);

    public event RideRequestEventHandler OnNewRideRequest;

    public void AddNewRequest(RideRequest request)
    {
        Console.WriteLine($"Hệ thống nhận yên cầu từ {request.CustomerName}");
        OnNewRideRequest?.Invoke(this, new RideRequestEventArgs(request));
    }
}

public class RideRequestEventArgs : EventArgs
{
    public RideRequest Request { get; }
    public RideRequestEventArgs(RideRequest request) => Request = request;
}

// === SUBSCRIBER ===
public class Driver
{
    public string Name { get; set; }

    public void OnRideRequested(object sender, RideRequestEventArgs e)
    {
        Console.WriteLine($"Tài xế {Name} nhận được yêu cầu từ {e.Request.CustomerName}");
    }
}

// === ĐĂNG KÝ ===
var dispatcher = new RideDispatcher();
var driver1 = new Driver { Name = "An" };
var driver2 = new Driver { Name = "Bình" };

dispatcher.OnNewRideRequest += driver1.OnRideRequested;
dispatcher.OnNewRideRequest += driver2.OnRideRequested;

dispatcher.AddNewRequest(new RideRequest { CustomerName = "Khách A" });

OOP thể hiện
- Tính chất: Đa hình, nhiều Driver cùng phản ứng khác nhau.
- Đóng gói: Event che giấu delegate bên trong, ngoài chỉ được += hoặc -=.
- Trừu tượng: RideDispatcher không cần biết ai đang lắng nghe.

Serialization (PACK/UNPACK) & IDisposable

Serialization
- GetObjectData(SerializationInfo info, StreamingContext context)
  PACK – Xếp đồ: Bạn tự tay bỏ từng món đồ (thuộc tính) vào vali, kèm nhãn dán (key).
  Ví dụ: info.AddValue("ten", this.Ten);

- Constructor đặc biệt: Student(SerializationInfo info, StreamingContext context)
  UNPACK – Bốc đồ: Nhận cái vali từ bưu tá, mọi từng món ra dựa theo nhãn.
  Ví dụ: Ten = info.GetString("ten");

IDisposable
- Quản lý tài nguyên (file, network). Trong Grab, khi đọc/ghi file serialize, cần đóng FileStream.
- Sẽ dùng using hoặc tự implement.
- Nhấn mạnh: đóng gói (encapsulation) logic giải phóng tài nguyên.

QUAN HỆ (Relationships)

1. Inheritance (Kế thừa) – Quan hệ "Is-a"
- Lớp con là một phiên bản cụ thể của lớp cha.
- Cho phép tái sử dụng code và đạt đa hình (Polymorphism) lúc runtime.

2. Composition (Hợp thành) – Quan hệ "Has-a" mạnh
- Sở hữu chặt chẽ.
- Phần là không thể thiếu của toàn thể.
- Nếu đối tượng chứa bị hủy, đối tượng thành phần cũng bị hủy theo.

3. Aggregation (Tập hợp) – Quan hệ "Has-a" yếu
- Sở hữu lỏng lẻo.
- Đối tượng chứa có thể chứa một nhóm đối tượng khác, nhưng các đối tượng đó có thể tồn tại độc lập.

4. Association (Liên kết) – Quan hệ "Uses-a" lâu dài
- Một đối tượng biết về một đối tượng khác và có thể tương tác với nó.

5. Dependency (Phụ thuộc) – Quan hệ "Uses-a" tạm thời
- Quan hệ yếu nhất và ngắn hạn nhất.
- Một lớp A phụ thuộc lớp B nếu B được sử dụng như tham số trong method của A, hoặc là biến cục bộ trong A.
