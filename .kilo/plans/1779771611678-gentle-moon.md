# Kế hoạch cải thiện tài liệu mã nguồn (Documentation Improvement Plan)

**Ngày:** 2026-05-26 12:00
**Mục tiêu:** Cập nhật tài liệu (`FINAL.md`, `README.md`, `REPORT.md`) để người đọc có thể hiểu toàn bộ logic code mà không cần mở code.

## Các bước thực hiện

### 1. Phân tích và Audit tài liệu hiện tại
- [ ] Rà soát `FINAL.md` để đối chiếu với cấu trúc code thực tế (Tầng Domain, Application, Infrastructure, Presentation).
- [ ] Rà soát `REPORT.md` để đảm bảo các kịch bản thực thi và sơ đồ khớp với code.
- [ ] Kiểm tra các file `UserControl` chính để trích xuất logic nghiệp vụ quan trọng (State transitions, Event handling).

### 2. Cập nhật tài liệu kỹ thuật chi tiết (`FINAL.md`)
- [ ] Thêm phần giải thích về luồng khởi tạo ứng dụng (Dependency Injection thủ công trong `AppServiceBundle`).
- [ ] Chi tiết hóa logic xử lý trong các Service chính (ví dụ: luồng Matching trong `MatchingService`, luồng tính giá trong `FareService`).
- [ ] Mô tả chi tiết các State Machine của `Trip` và `Driver` (Điều kiện chuyển trạng thái, các tác vụ thực hiện khi chuyển trạng thái).
- [ ] Cập nhật sơ đồ UI Layout để phản ánh chính xác các `Panel` và `UserControl` được sử dụng.

### 3. Cải thiện tài liệu hướng dẫn và báo cáo (`README.md`, `REPORT.md`)
- [ ] Cập nhật `README.md` với cái nhìn tổng quan nhanh và hướng dẫn chạy nhanh.
- [ ] Cập nhật `REPORT.md` phần "Phân tích và thiết kế lớp" để mô tả sâu hơn về hành vi (behavior) thay vì chỉ liệt kê thuộc tính.

### 4. Kiểm chứng và Retrospective
- [ ] Tự kiểm tra: Đọc tài liệu đã sửa và thử trả lời các câu hỏi về logic code (ví dụ: "Làm sao để biết tài xế nào được chọn cho chuyến đi?").
- [ ] Ghi nhận vào `agent/retrospective.md`.

## Kiểm chứng
- Tài liệu phải mô tả được:
  - Cách dữ liệu được lưu trữ và nạp lại.
  - Luồng di chuyển của một `Trip` từ lúc tạo đến lúc kết thúc.
  - Cách các thành phần UI giao tiếp với nhau (Event-driven).
  - Cách xử lý tranh chấp (Thread-safety) trong hệ thống.

## Rủi ro
- Tài liệu quá dài dòng gây khó đọc (Mức độ: Trung bình).
- Tài liệu bị lạc hậu so với code nếu code thay đổi sau này (Mức độ: Thấp - do đang sửa code theo tài liệu hoặc ngược lại).

## Nguồn tham khảo
- Mã nguồn hiện tại trong thư mục `OOP2026/`.
- Tài liệu `AGENTS.md` về quy tắc viết tài liệu.
