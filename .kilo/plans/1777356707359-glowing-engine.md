## 2. LocationPickerControl – Smart Suggestions & History
**Target**: `Presentation.Components.LocationPickerControl` (used by `UcPassenger`)

### 2.1 Redesign as ComboBox with OwnerDrawFixed
- Giữ tên class `LocationPickerControl` (không đổi để tránh thay đổi Designer references)
- **Implementation**: Đổi từ kế thừa `UserControl` sang kế thừa `ComboBox`.
  - **Lưu ý**: Việc đổi base class từ UserControl → ComboBox có thể phức tạp vì Designer file đã generated code cho UserControl. 
  - **Alternative**: Thay vì đổi base class, giữ `UserControl` nhưng chứa một `ComboBox` ẩn (dropdown) bên trong, và mô phỏng dropdown behavior. Tuy nhiên, để đơn giản và tận dụng `OwnerDrawFixed`, ta nên tạo **class mới** kế thừa `ComboBox` với tên `ucLocationPicker` (theo chuẩn `uc` prefix). Sau đó thay thế tất cả occurrences của `LocationPickerControl` trong UcPassenger.Designer bằng `ucLocationPicker`. 
  - **Decision**: Tạo `ucLocationPicker` mới (UserControl với ComboBox bên trong) để giữ full control over design, tránh rủi ro Designer. Thực tế: Một UserControl chứa:
    - Một Label để hiển thị text (ví dụ "A: Điểm đón")
    - Một ComboBox ẩn (DropDown) khi click vào control
    - Xử lý OwnerDrawFixed cho ComboBox để vẽ Header và items.
- **Plan**: Keep `LocationPickerControl` as is but implement new features as extension? Hmm.

Let's re-evaluate: The requirement is to add smart suggestions, history, fixed locations, and duplicate detection. The simplest approach is to convert `LocationPickerControl` into a ComboBox-derived control with custom drawing, and update references accordingly. This changes two files:
- `LocationPickerControl.cs` → rewrite as `public class LocationPickerControl : ComboBox`
- `UcPassenger.Designer.cs`: no change needed because the control type remains same name; Designer code will work if we keep same namespace and class name. But the generated code expects it to be a UserControl? Actually Designer code declares `this.pickupPicker = new Presentation.Components.LocationPickerControl();`. If we change base class from UserControl to ComboBox, the Designer code may still compile because ComboBox is a Control. But the Designer file might have initialization specific to UserControl (like setting `AutoSize`, etc.). We need to inspect.

## 2. UcPassenger – LocationPicker & Booking Validation

### 2.1 LocationPickerControl – Smart Suggestions & History
**Target**: `Presentation.Components.LocationPickerControl` (used by `UcPassenger`)

#### Approach: Rewrite as ComboBox-derived with OwnerDrawFixed
- Keep the existing class name `LocationPickerControl` to avoid modifying Designer references.
- Change base class from `UserControl` to `ComboBox`.
  - Set properties in constructor/InitializeComponent:
    - `DropDownStyle = ComboBoxStyle.DropDownList`
    - `DrawMode = DrawMode.OwnerDrawFixed`
    - `ItemHeight = 28`
    - `DropDownWidth = 300` (enough for address text)
- Remove the existing `OnPaint` override (ComboBox draws itself).
- Implement custom drawing via `OnDrawItem`:
  - Maintain an internal list `List<DropdownItem> _items` where `DropdownItem` has `bool IsHeader`, `string HeaderText`, `Location Location`.
  - When drawing, if `IsHeader`: bold font, light gray background, centered text.
  - Otherwise: normal text showing address (District, City).
- Expose `SelectedLocation` that gets set when a non-header item is selected.
- Add properties: `public List<Location> RecentLocations { get; }`, `public List<Location> FixedLocations { get; }`, `public Location CurrentPickup { get; set; }`, `public Location CurrentDestination { get; set; }`.
- Build dropdown content via method `PopulateDropdown()` that inserts Header items ("GẦN ĐÂY", "CƠ SỞ UEH") followed by location items.
- Recent locations stored in static `Dictionary<string, List<Location>>` keyed by passenger phone or ID; load/save to JSON in `%APPDATA%\RideGo\recent.json`.
- Fixed UEH campuses: predefined Location objects (Main, Thu Duc, An Giang) always included.

### 2.2 UcPassenger Booking Validation
- Add `ErrorProvider _errorProvider` to `UcPassenger`.
- Add validation method `ValidateBooking()`:
  - pickupSelected != null
  - destinationSelected != null
  - pickup ≠ destination (tolerance)
  - vehicle type selected (cmbVehicleType.SelectedIndex >= 0)
- Call on `pickupPicker.SelectedIndexChanged`, `destinationPicker.SelectedIndexChanged`, `cmbVehicleType.SelectedIndexChanged`.
- Set error messages accordingly; enable `btnBook` only if all valid.
- Show ErrorProvider icons with tooltips.

### 2.3 Button hover effects in UcPassenger
- `btnBook`, `btnCancelSearch`, `btnCancelTrip`, `btnConfirmPayment`, `btnRateDriver`: add MouseEnter/Leave.
- Define colors (primary, danger, success) using constants or a theme helper.
- Use `FlatAppearance` for border changes if desired.

### 2.4 Keyboard support in UcPassenger
- Set `KeyPreview = true` on `UcPassenger`.
- Handle `KeyDown`:
  - Enter: if focus is on `pickupPicker`/`destinationPicker`/`cmbVehicleType` → ignore (let default).
  - Else if `btnBook.Enabled` → `btnBook.PerformClick()`.
  - Else if focus on other controls → `SelectNextControl`.
- `e.SuppressKeyPress = true` when Enter triggers navigation/submit.

## 6. Edge Cases & Considerations

- **ComboBox Enter behavior**: When a ComboBox has focus, Enter opens dropdown by default. Suppress if needed.
- **ErrorProvider alignment**: For custom ComboBox, use `SetIconAlignment(ctrl, ErrorIconAlignment.MiddleRight)`.
- **Persistence**: JSON storage should handle I/O failures gracefully (fallback to memory).
- **Theme consistency**: Hover colors from `UiConstants`.
- **Accessibility**: Ensure color contrast and readable tooltips.
