# Plan: Fix Mock Classes to Properly Implement Interfaces

## Overview
This plan outlines the necessary changes to fix mock classes that are not fully implementing their respective interfaces. Each mock class needs to implement all methods defined in its interface with correct signatures and return types.

## Issues Identified

### 1. MockMapService.cs (Implements IMapService)
**Missing Methods:**
- `Task<List<Location>> SearchLocationAsync(string query)`
- `Task<Location> ReverseGeocodeAsync(double latitude, double longitude)`
- `Task<Route> GetRouteAsync(Location start, Location end)`

**Extra Methods (Not in Interface):**
- `Task<Route> CalculateRouteAsync(Location origin, Location destination, Domain.Enums.VehicleType vehicleType)`
- `Task<Location> GeocodeAddressAsync(string address)`
- `Task<string> GetAddressFromCoordinateAsync(Coordinate coordinate)`
- `Task<double> CalculateDistanceAsync(Coordinate origin, Coordinate destination)`
- `Task<System.TimeSpan> CalculateDurationAsync(Coordinate origin, Coordinate destination)`

### 2. MockUserService.cs (Implements IUserService)

- `Task<Driver> GetDriverByIdAsync(Guid driverId)`
- `Task<Passenger> GetPassengerByIdAsync(Guid passengerId)`

- `Task UpdateAsync(Vehicle entity)`

## Fix Approach
For each mock class:
1. Remove extra methods/events not present in the interface
2. Add missing methods with correct signatures matching the interface
3. Implement methods to return appropriate mock data using `Task.FromResult()` or `Task.CompletedTask`
4. For methods returning collections, return empty lists or mock data as appropriate
5. For methods returning single objects, return mock instances or null as appropriate
6. Ensure all implementations are thread-safe and suitable for UI testing

## Implementation Details
Each mock method should:
- Track call counts/flags for verification in tests
- Return predictable mock data to enable consistent test behavior
- Be configurable via setup methods for different test scenarios
- Follow the existing pattern in the codebase for mock implementations
