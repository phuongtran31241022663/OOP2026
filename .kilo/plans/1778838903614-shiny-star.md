# Kế hoạch: Fix DPI / layout cho form chứa password + eye icon

## Mục tiêu
- Đảm bảo layout ở runtime khớp với Designer trên các DPI/scale (100%, 125%, 150%).
- Icon "mắt" giữ kích thước và vị trí (không bị co nhỏ hoặc control bị kéo cao).
- Tránh TableLayoutPanel autosize làm thay đổi chiều cao row runtime.

## Giả định
- Project là WinForms (Designer + Designer.cs tồn tại).
- Hiện đang dùng `AutoScaleMode = Font` và Windows scale ≠ 100%.
- Password row đang nằm trong `TableLayoutPanel` và row đó đang AutoSize/Percent.

## Tổng quan giải pháp (cao cấp)
1. Chọn chế độ AutoScale hợp lý cho Form (None hoặc Dpi).  
2. Không để row password auto-size: dùng SizeType = Absolute với height cố định.  
3. Tách TextBox + Eye button vào một Panel (txt Dock=Fill, button Dock=Right, fixed width).  
4. Fix property của button/picture (ImageAlign / BackgroundImageLayout / SizeMode).  
5. Nếu cần DPI-aware: bật manifest PerMonitorV2 và cung cấp assets đa kích thước hoặc vector.

## Các bước chi tiết (thực hiện tuần tự)

1) Audit (10–30 phút)
- Tìm toàn bộ form/control có `AutoScaleMode = AutoScaleMode.Font`.
- Tìm TableLayoutPanel có row AutoSize / Percent dùng cho password.
- Tìm control eye: tên thường `btnShowPassword`, `pictureBoxEye` hoặc tương tự.
- Tools: VS Find in Files (AutoScaleMode, TableLayoutPanel, RowStyles, BackgroundImageLayout, ImageAlign).

2) Quyết định AutoScaleMode (5–15 phút)
- Nếu cần pixel-stable (match Designer): set
```csharp
this.AutoScaleMode = System.Windows.Forms.AutoScaleMode.None;
```
- Nếu muốn app scale theo DPI (phù hợp nhiều màn hình DPI khác nhau): set
```csharp
this.AutoScaleMode = System.Windows.Forms.AutoScaleMode.Dpi;
```
  và bật DPI awareness trong app.manifest (PerMonitorV2) nếu target Windows hiện đại.

3) Fix TableLayoutPanel (10–30 phút)
- **QUAN TRỌNG**: Phải add tất cả RowStyles trước, sau đó mới modify:
```csharp
// Add all rows first
this.pnlLogin.RowStyles.Add(new System.Windows.Forms.RowStyle(System.Windows.Forms.SizeType.Absolute, 47F));
this.pnlLogin.RowStyles.Add(new System.Windows.Forms.RowStyle(System.Windows.Forms.SizeType.Absolute, 28F));
this.pnlLogin.RowStyles.Add(new System.Windows.Forms.RowStyle(System.Windows.Forms.SizeType.Absolute, 38F));
this.pnlLogin.RowStyles.Add(new System.Windows.Forms.RowStyle(System.Windows.Forms.SizeType.Absolute, 28F));
this.pnlLogin.RowStyles.Add(new System.Windows.Forms.RowStyle(System.Windows.Forms.SizeType.Absolute, 38F)); // row 4 - password
// ... add remaining rows

// THEN modify the password row to fixed height
this.pnlLogin.RowStyles[4].SizeType = System.Windows.Forms.SizeType.Absolute;
this.pnlLogin.RowStyles[4].Height = 40F;
```
- Đảm bảo `tblMain.AutoSize = false` cho panel chính nếu cần.
- Tránh `Dock = Fill` + `AutoSize = true` trên cùng control nếu không muốn autoshrink/grow.

4) Tách TextBox + Eye vào Panel (15–45 phút)
- Tạo `panelPassword` dock = Fill.
- Thêm `txtPassword.Dock = DockStyle.Fill`.
- Thêm `btnEye.Dock = DockStyle.Right; btnEye.Width = 40;`.
- Ví dụ code (có thể đặt trong Designer hoặc runtime):
```csharp
panelPassword.Dock = DockStyle.Fill;

txtPassword.Dock = DockStyle.Fill;

btnEye.Width = 40;
btnEye.Dock = DockStyle.Right;
btnEye.BackgroundImage = Properties.Resources.eye_24; // hoặc Image
btnEye.BackgroundImageLayout = ImageLayout.Zoom;
btnEye.ImageAlign = ContentAlignment.MiddleCenter;

panelPassword.Controls.Add(txtPassword);
panelPassword.Controls.Add(btnEye);
```
- Lợi: layout ổn định, không phụ thuộc row autosize của TableLayoutPanel.

5) Chỉnh properties cho icon/button (5–15 phút)
- Nếu dùng Button.Image: `btn.ImageAlign = ContentAlignment.MiddleCenter; btn.TextImageRelation = TextImageRelation.ImageBeforeText;`.
- Nếu dùng BackgroundImage: `btn.BackgroundImageLayout = ImageLayout.Zoom;`.
- Nếu dùng PictureBox: `pictureBox.SizeMode = PictureBoxSizeMode.Zoom;`.
- Cân nhắc dùng 24x24 (hoặc 32x32) assets và `Zoom` để khử lệch.

6) Font cố định (5 phút)
- Nếu Designer/runtime dùng font khác nhau, set explicit font cho Form:
```csharp
this.Font = new System.Drawing.Font("Segoe UI", 9F);
```

7) Tránh mix Anchor + Dock (5–10 phút)
- Trong panel nhỏ này chỉ dùng `Dock` cho txt và button; không dùng Anchor để tránh behavior bất ngờ.

8) Refactor layout lớn nếu cần (30–90 phút)
- Nếu repo dùng nhiều nested TableLayoutPanel gây bất ổn ở DPI, chuyển sang Main Panel + FlowLayoutPanel/stacked panels với fixed widths/heights.
- Ví dụ: MainPanel (Dock=Fill) -> CenterPanel (fixed Width) -> FlowLayoutPanel (vertical).

9) (Tuỳ chọn) DPI-awareness nâng cao
- Nếu chọn `AutoScaleMode.Dpi`, thêm app.manifest với PerMonitorV2 và cung cấp assets 1x/1.5x/2x hoặc dùng vector (SVG) library.
- Thời gian thêm manifest + test: 10–30 phút.

## Kiểm thử (test matrix)
- Test scale: 100%, 125%, 150% trên cùng máy.
- Test per-monitor: kéo app giữa màn hình 100% và 150% (nếu bật PerMonitorV2).
- Trong Visual Studio Designer: so sánh render với runtime.
- Kiểm tra tiêu chí chấp nhận:
  - Chiều cao row password không đổi giữa Designer và runtime.
  - Icon mắt có kích thước + căn giữa giống mong đợi.
  - Không có gap/white-space bất ngờ do TLP expand.

## Files cần chỉnh (dự kiến)
- FormName.Designer.cs: thay `AutoScaleMode`, RowStyles, move controls vào Panel, chỉnh property button.
- FormName.cs: chỉnh font runtime nếu cần.
- app.manifest (tuỳ chọn): enable PerMonitorV2.
- Resources: thêm image eye ở kích thước phù hợp.

## Acceptance criteria
- Trên các scale kiểm thử, view password row cao và icon mắt giống nhau, không co nhỏ.
- Layout không bị kéo dãn theo chiều dọc bất ngờ.
- Designer và runtime tương đồng về kích thước control.

## Ước lượng thời gian
- Audit + small fixes: 0.5–1.5 giờ
- Refactor layout lớn (nếu cần): 1–3 giờ
- Thử nghiệm cross-DPI: 0.5–1 giờ

## Rủi ro / Ghi chú
- `AutoScaleMode.None` làm mất auto-scaling cho font trên high-DPI — nếu app cần scale chữ, chọn `Dpi` và xử lý assets.
- Nếu target nhiều máy với DPI khác nhau, khuyến nghị PerMonitorV2 + multi-resolution assets hoặc vector icons.
- **LỖI QUAN TRỌNG**: Khi sửa Designer.cs, phải đảm bảo thứ tự đúng:
  1. Khai báo field Panel ở đầu file
  2. Khởi tạo Panel trong InitializeComponent (new Panel())
  3. Add tất cả RowStyles vào TableLayoutPanel
  4. SAU ĐÓ mới modify RowStyles[index]
  5. Cuối cùng mới add Panel vào TableLayoutPanel.Controls
- Nếu gặp lỗi "Index was out of range", kiểm tra xem có đang modify RowStyles[index] trước khi Add row đó chưa.
